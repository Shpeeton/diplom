## 2. Перечень используемых инструментов

| Инструмент | Обоснование |
|------------|-------------|
| Python | Язык программирования для написания автотестов |
| pytest | Фреймворк для запуска тестов, удобные фикстуры и параметризация |
| Selenium | Библиотека для автоматизации действий в браузере |
| Allure | Фреймворк для генерации наглядных отчетов с шагами и скриншотами |
| Docker | Контейнеризация для одновременного запуска БД, эмулятора и приложения |
| pymysql | Драйвер для подключения к MySQL и проверки данных в БД |
| psycopg2-binary | Драйвер для подключения к PostgreSQL |
| Git | Хранение кода и документации в репозитории |

## 3. Риски при автоматизации

| Риск | Способ минимизации |
|------|-------------------|
| Docker не запускается | Проверить установку Docker Desktop, освободить порты 3306, 8080, 8081, 9999 |
| Gate-simulator не стартует | Запустить локально через Node.js, как указано в задании |
| Ошибки подключения к БД | Использовать host.docker.internal для доступа к локальному эмулятору |
| Тесты падают из-за таймингов | Использовать WebDriverWait с увеличенным таймаутом |
| Allure не генерирует отчет | Добавить инструкцию по установке allure-commandline в README |
| Разные БД (MySQL и PostgreSQL) | Запустить два экземпляра приложения на разных портах |

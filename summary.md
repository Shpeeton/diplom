# Отчет о проведенной автоматизации

## Что было запланировано и что реализовано

### Запланировано:
- Настройка Docker окружения (MySQL, PostgreSQL, gate-simulator, приложение)
- Написание Page Object моделей
- Написание автотестов (позитивные и негативные сценарии)
- Поддержка двух баз данных (MySQL и PostgreSQL)
- Интеграция Allure для отчетов
- Оформление документации (Plan.md, Report.md, Summary.md, README.md)

### Реализовано:
Все пункты плана выполнены в полном объеме.

## Причины, по которым что-то не было реализовано
Все запланированные сценарии реализованы. Отклонений нет.

## Сработавшие риски

| Риск | Как проявился                     | Способ минимизации                         |
|------|-----------------------------------|--------------------------------------------|
| Docker не запускается | Конфликты портов (3306, 8080)     | Освобождены порты, перезапущены контейнеры |
| Тесты падают из-за таймингов | Уведомления не успевали появиться | Увеличено время ожидания до 11 секунд      |
| Allure не генерирует отчет | Не установлен allure на ПК        | Добавлена инструкция в README              |

## Интеграция Allure

- **allure-pytest** добавлен в зависимости
- В `conftest.py` реализовано прикрепления скриншотов в отчет allure при падении тестов
- Результаты тестов сохраняются в директорию `allure-results`
- Для просмотра отчета используется команда `allure serve allure-results`
